# npm 版本问题

  • 编译阶段： npm i / yarn 安装的 npm 包不一致，导致 bpm 包版本误差，而编辑不通过
  • 部署阶段： docker 打镜像时，会有 package.json 缓存 npm 包，也会导致打包后的 egg 项目执行出错
  • 解决思路： 需要清空环境，从零开始。

# 引入的包之间相互依赖
